# Playbook: why & How — Nucleus Hyper-Industrial Logic

This manual explains the BIT-BY-BIT logic of the Nucleus module and how it has been evolved to "Hyper-Industrial" status via nodal state machines and multi-memory synthesis.

## 1. 🔍 Nodal Deconstruction (State Machine 1.x)
- **The Bit**: We break deconstruction into 4 discrete micro-states defined in `sub_graphs.json`.
- **Industrial Improvement**: Instead of a single pass, the agent moves through **1.1 Extraction** -> **1.2 Cleanse** -> **1.3 Validation**. This ensures that "Analogy Contamination" is physically removed before the strategy is built.

## 2. 🌲 Multi-Memory Expansion (Trajectories A & B)
- **The Bit**: Parallel reasoning paths (Tree of Thoughts) now share a `live_notes` buffer.
- **Industrial Improvement**: As Trajectory A (Speed) discovers a risk, it "broadcasts" a note to the shared memory. Trajectory B (Durability) then adapts its logic mid-loop to account for that risk.

## 3. 🌉 The Master-State Synthesis (The Combination Bridge)
- **The Bit**: A dedicated "Combinator" flow that merges the winners from each trajectory.
- **Industrial Improvement**: The Nucleus no longer forces a choice between "Fast" and "Durable." It creates a hybrid "Fast-Moat" strategy by resolving logical trade-offs in sub-state **5.2**.

## 4. 📈 Recursive Sub-Graphs (Execution Integrity)
- **The Bit**: Every operation has a nested subgraph (Sub-States).
- **Industrial Improvement**: Sub-states must return a `validation_hash`. If the hash doesn't match the expected "Perfect Execution" pattern, the state machine recurses to the previous node with a self-correction note.

## ⚡ Hyper-Industrial Signatures (Multi-Memory I/O)
- **The Bit**: Expanded JSON shells including `sub_state_id` and `live_notes`.
- **Benefit**: 100% observability of the agent's "improvisation" steps and mid-task notes, leading to zero-error strategy synthesis.
