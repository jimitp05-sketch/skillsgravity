# Minor Skill: Recursive Deep Brainstormer

Use this skill in the `anti-gravity-nucleus` phase to ensure no detail is missed before planning.

## 🛠️ Step-by-Step Protocol

### 1. The Strategic "What"
- **Action**: Identify the core goal, user pain point, and success metric.
- **Question**: "If this succeeds perfectly, what does the world look like?"

### 2. The Architectural "How"
- **Action**: Decompose the goal into the 7-Module Engine categories.
- **Recursion**: For each module, ask: "What specific skills or data are required here?"

### 3. Constraint Stress-Testing
- **Action**: Identify what could go wrong (Token limits, API costs, Security risks).
- **Resolution**: Adjust the architecture to mitigate these early.

### 4. The Final Synthesis
- **Action**: Output the **Master Design Doc**.
- **Exit**: Move directly to `anti-gravity-systems/minor-planning`.
